# 14-reply-to-reply-low

**As a** logged-in user  
**I want to** reply to a reply  
**So that** I can continue nested discussions at any level

**Priority:** Low (Not required for initial release)

**Acceptance Criteria:**
- Reply button is available **only** to logged-in users on replies
- For non-logged-in visitors, reply button shows "Login to reply" prompt
- User can reply to replies creating deeper nesting
- Replies maintain visual hierarchy with indentation
- System supports multiple levels of nesting
- All replies are displayed in chronological order within their thread
- Nested structure is mobile responsive
- **Login is required** to post replies (viewing nested replies does not require login)
